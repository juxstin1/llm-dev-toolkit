# Spec0 Plan

Create a Spec0 plan for:

`$ARGUMENTS`

## Output

Write a concise implementation frame with:

- Goal: the concrete outcome to produce.
- Context: relevant repo areas, existing patterns, and instructions that matter.
- Non-goals: tempting work that should stay out of scope.
- Assumptions: decisions made because the prompt or repo is incomplete.
- Plan: ordered slices small enough to verify independently.
- Verification: commands, tests, manual checks, or review passes that prove the work.
- Risks: any edge cases or migration concerns to watch.

Do not edit files unless the user asked you to implement after planning. If implementation is expected, continue after the plan.
